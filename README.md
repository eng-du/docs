# 잉듀는 어떤 서비스인가요?

> **Eng**lish + e**du**cation = **Engdu**

잉듀는 생성형 AI를 활용한 `영어 독해 학습 서비스`에요. 원하는 주제로 영어 지문과 퀴즈를 생성하여, 영어 독해를 어려워하는 초보자 분들도 재미있게 학습을 시작할 수 있어요.

<br>

# 왜 이런 서비스를 만들게 됐나요?

`영어 학습을 재미있게` 하고 싶어서 만들었어요. 

개발 공부를 하기 위해 공식 문서를 찾다 보면, 대부분의 문서가 영어로 작성되어 있어 원하는 정보를 찾기 어렵고, 기술적 내용보다 언어 자체가 벽으로 느껴질 때가 있었어요.

영어 독해에 대한 필요성을 느끼고 공부를 시작했지만, 기존의 학습 방식에서 아래 문제점들을 발견했어요.

- **관심 없는 주제**: 독해 지문의 주제가 재미없어요.
- **높은 진입 장벽**: 초보자에게 너무 어려운 난이도로 금방 지쳐서 공부를 그만두게 돼요.

잉듀 개발팀은 이 문제점들을 소프트웨어적으로 해결하고 싶었어요. 

생성형 AI로 학습자가 읽고 싶은 주제의 지문을 읽으면 **학습 경험이 보다 재밌을** 거라 기대했어요. 또, 지문을 읽는 과정에서 학습을 도울 수 있는 **인터랙티브한 요소**가 있다면 **초보자의 학습 허들을 낮추고** 몰입감있게 독해 공부를 할 수 있을 거라 생각했어요.

<br>

# 기능을 소개합니다!

### 1️⃣ 영어 학습 주제 입력

사용자는 원하는 **주제나 키워드**를 입력하여 영어 학습을 시작할 수 있어요.  
입력된 내용을 기반으로 AI가 **영어 학습 콘텐츠를 자동 생성해요.**

![input](https://github.com/user-attachments/assets/4ecd5ffe-6f8b-4973-a20b-f0d4b38b9370)

### 2️⃣ 생성 대기 중 구동사 학습

영어 학습 콘텐츠가 생성되는 데 약 **1분 정도의 시간이 소요돼요.**  지루할 수 있는 대기 시간을 활용해 **구동사**를 먼저 학습할 수 있도록 했어요.  
구동사의 의미와 예문을 통해 **자주 사용되는 영어 표현을 자연스럽게 익힐 수 있어요.**

![phrasal](https://github.com/user-attachments/assets/6cf0dbbd-cf0b-4a6d-9260-b84adf978dfc)

### 3️⃣ 청크 기반 독해

문장을 **청크 단위로 분리**하여 영어 문장을 단계적으로 이해할 수 있도록 도와요.  
클릭을 통해 **원문과 해석을 번갈아 확인**하며 직독직해 방식의 독해 연습을 할 수 있어요.

![chunk](https://github.com/user-attachments/assets/5117c7e6-64a3-42ca-bf08-063ae8942240)

### 4️⃣ 이해도 확인 퀴즈

지문을 읽은 뒤 **퀴즈를 통해 학습 내용을 점검할 수 있어요.**
문제는 **독해(내용 일치/불일치), 문법, 어휘(유의어/반의어)** 로 구성되어 있어요.  
정답 여부와 함께 해설이 제공되어 **학습 내용을 다시 복습할 수 있어요.**

![quiz](https://github.com/user-attachments/assets/d1b1dd05-0b6e-4b05-aee9-47f53e174dd6)

# 잉듀 개발팀의 기술적 고민을 소개합니다.

서비스를 만들어가는 과정에서 마주한 다양한 기술적 고민과 해결의 흔적을 기록합니다. 이외의 더 자세한 경험들은 [위키](https://github.com/eng-du/docs/wiki)에서 확인할 수 있습니다.

### **[BE] LLM 기반 콘텐츠 생성에서 발생한 중복 생성 문제를 상태 머신, DB 락으로 해결하기**

LLM API 호출 시간이 긴 환경에서 발생한 콘텐츠 중복 생성 문제를 해결하기 위해 상태 머신 기반 구조를 도입했습니다. 이 문제를 해결하기 위해 멱등 키, DB 유니크 제약조건, 비관적 락, 트랜잭션 길이 등 다양한 요소를 고려했습니다.

> 작성자: 박영규
> 
> 
> 🔗 [링크](https://github.com/eng-du/docs/wiki/%EC%83%81%ED%83%9C-%EB%A8%B8%EC%8B%A0%EA%B3%BC-DB-%EB%9D%BD%EC%9D%84-%EC%9D%B4%EC%9A%A9%ED%95%B4-LLM-%EC%BD%98%ED%85%90%EC%B8%A0-%EC%A4%91%EB%B3%B5-%EC%83%9D%EC%84%B1-%EB%B0%A9%EC%A7%80%ED%95%98%EA%B8%B0)
> 

### **[BE] AWS SQS 기반 비동기 워커로 LLM 장애 격리 및 장애 회복하기**

불안정한 외부 LLM API의 장애로 인한 작업 유실 문제를 해결하기 위해, 요청 처리와 작업 수행을 분리하는 SQS 기반 비동기 워커 구조를 설계한 내용입니다. **Visibility Timeout, DLQ, retryAllowedAt** 등을 활용해 작업 유실 방지와 재처리를 구현하여 서비스 안정성과 장애 복구 능력을 향상시켰습니다.

> 작성자: 박영규
> 
> 
> 🔗 [링크](https://github.com/eng-du/docs/wiki/AWS-SQS-%EA%B8%B0%EB%B0%98-%EB%B9%84%EB%8F%99%EA%B8%B0-%EC%9B%8C%EC%BB%A4%EB%A1%9C-LLM-%EC%9E%A5%EC%95%A0-%EA%B2%A9%EB%A6%AC-%EB%B0%8F-%EC%9E%A5%EC%95%A0-%ED%9A%8C%EB%B3%B5%ED%95%98%EA%B8%B0)
> 

### **[BE] 아웃 박스 패턴으로 메시지 발행과 DB 정합성을 보장하기**

LLM 기반 콘텐츠 생성 과정에서 DB 트랜잭션과 메시지 발행이 서로 다른 시스템에서 처리되면서, 메시지가 먼저 처리되거나 트랜잭션이 롤백되었음에도 메시지가 발행되는 일관성 문제가 발생했습니다. 이로 인해 예상치 못한 예외가 증가하고, 사용자 대기 시간이 늘어나는 등 서비스 품질에 영향을 미치는 문제가 확인되었습니다.

이 글에서는 이러한 문제를 해결하기 위해 **Transactional Outbox 패턴**을 도입한 과정을 설명합니다.

> 작성자: 박영규
